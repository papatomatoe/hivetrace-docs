## Назначение

HiveTrace Gateway (LiteLLM + HiveTrace) выступает прокси-уровнем между вашим приложением и LLM-провайдером.
Вместо прямого обращения к OpenAI или другому провайдеру приложение вызывает Gateway, который обеспечивает маршрутизацию, мониторинг и применение политик.

Gateway:

- маршрутизирует запросы к моделям через LiteLLM
- анализирует входящие и исходящие сообщения
- применяет политики безопасности
- обнаруживает и маскирует чувствительные данные
- связывает события с приложением, пользователем и сессией
- сохраняет телеметрию для аудита и расследований

Gateway предоставляет **OpenAI-совместимый API**, поэтому интеграция требует минимальных изменений клиентского кода.

---

## Архитектура взаимодействия

Типовой поток:

```
Application → Gateway (LiteLLM) → LLM
                     ↓
                  HiveTrace
```

1. Приложение отправляет запрос в Gateway.
2. Gateway передаёт его в модель через LiteLLM.
3. HiveTrace анализирует входные и выходные данные.
4. В зависимости от конфигурации Gateway может:
   - разрешить ответ,
   - модифицировать его,
   - заблокировать его,
   - зафиксировать событие.

## Требования к интеграции

Для работы через Gateway приложение должно:

- использовать `base_url` уровня `/v1`
  пример:

  ```
  http://localhost:4100/v1
  ```

- передавать `Authorization: Bearer <LITELLM_MASTER_KEY>`
- передавать `X-Application-Id` (UUID приложения HiveTrace)
- передавать идентификаторы пользователя и сессии

Gateway совместим с OpenAI SDK и любыми HTTP-клиентами, поддерживающими Chat Completions API.

---

## Режимы интеграции

HiveTrace Gateway поддерживает два основных режима:

1. Использование без мультиагентов
2. Использование с мультиагентами

Выбор зависит от архитектуры вашего приложения.
