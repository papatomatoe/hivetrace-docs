## Назначение

Раздел описывает формат тела запроса (`body`), который должен попадать на вход Gateway для корректной обработки сообщения с файлами в HiveTrace при работе через OpenWebUI или при обращении к сервису другим способом.

Стандартный ответ OpenWebUI не содержит в `metadata.files` поля `request_id`.
При интеграции через Gateway тело ответа модифицируется: добавляется `request_id`, а каждому объекту в `metadata.files` присваивается `file.request_id` для привязки приложенных файлов и изображений к конкретному сообщению.

---

## Поле `request_id`

В `metadata` добавляется поле `request_id`, значение которого зависит от направления сообщения:

| Источник сообщения     | Значение `request_id`           |
| ---------------------- | ------------------------------- |
| Ответ модели           | `message_id` текущего сообщения |
| Сообщение пользователя | `parent_message_id` сообщения   |

Это позволяет однозначно связать запрос с записью в HiveTrace.

---

## Привязка файлов и изображений

Все вложения - файлы и изображения - размещаются в массиве `metadata.files`.

Каждому объекту в `metadata.files` присваивается значение `file.request_id`, равное `request_id` сообщения.
Это обеспечивает поиск файлов, приложенных к конкретному сообщению, на стороне HiveTrace.

Изображения помещаются в тот же массив `metadata.files` и имеют ключевые поля аналогичные `file.request_id`.

---

## Структура `metadata.files`

### Файл

| Поле              | Тип      | Описание                                          |
| ----------------- | -------- | ------------------------------------------------- |
| `type`            | `string` | Тип вложения. Для файлов: `"file"`.               |
| `file.id`         | `string` | UUID файла.                                       |
| `file.filename`   | `string` | Имя файла.                                        |
| `file.meta`       | `object` | Метаданные: `name`, `content_type`, `size`.       |
| `file.request_id` | `string` | Идентификатор сообщения для привязки в HiveTrace. |
| `id`              | `string` | UUID файла (дублирует `file.id`).                 |
| `name`            | `string` | Отображаемое имя файла.                           |

### Изображение

| Поле              | Тип      | Описание                                          |
| ----------------- | -------- | ------------------------------------------------- |
| `type`            | `string` | Тип вложения: `"image"`.                          |
| `url`             | `string` | Содержимое в формате `data:<mime>;base64,...`.    |
| `name`            | `string` | Имя изображения.                                  |
| `mime_type`       | `string` | MIME-тип изображения.                             |
| `file.request_id` | `string` | Идентификатор сообщения для привязки в HiveTrace. |

## Пример тела запроса

```json
{
  "model": "gpt-5-web-search",
  "messages": [
    {
      "role": "user",
      "content": "Как создать файл?"
    }
  ],
  "stream": false,
  "metadata": {
    "user_id": "e9bacbd0-bc74-4452-aeb8-d45868e77135",
    "chat_id": "273f217f-bb0e-4735-bab8-e9522488584b",
    "message_id": "11ae3438-c513-402e-93d6-58a43ac7c0cb",
    "parent_message_id": "caa6da20-21df-4645-84e1-612633d92e8e",
    "session_id": "273f217f-bb0e-4735-bab8-e9522488584b",
    "files": [
      {
        "type": "file",
        "file": {
          "id": "c9a62239-e849-4391-9fac-09a8b9ab3715",
          "filename": "test.txt",
          "meta": {
            "name": "test.txt",
            "content_type": "text/plain",
            "size": 23
          },
          "request_id": "caa6da20-21df-4645-84e1-612633d92e8e"
        },
        "id": "c9a62239-e849-4391-9fac-09a8b9ab3715",
        "name": "test.txt"
      },
      {
        "type": "image",
        "url": "data:image/jpeg;base64,...",
        "name": "images11.jpeg",
        "mime_type": "image/jpeg",
        "file": {
          "request_id": "caa6da20-21df-4645-84e1-612633d92e8e"
        }
      }
    ],
    "request_id": "caa6da20-21df-4645-84e1-612633d92e8e"
  }
}
```

В данном примере `request_id` равен `parent_message_id`, так как контекст запроса - сообщение пользователя. Также важно, что на текущем этапе url изображения содержит не ссылку, а само содержание изображения в формате base64.

## Итог

При работе через Gateway с собственным OpenWebUI или иным клиентом:

1. в `metadata` добавляется `request_id` для привязки сообщения к записи в HiveTrace
2. значение `request_id` определяется направлением сообщения: `message_id` для ответов модели, `parent_message_id` для сообщений пользователя
3. все файлы и изображения размещаются в `metadata.files` с индивидуальным `file.request_id`
4. структура обеспечивает поиск вложений по конкретному сообщению на стороне HiveTrace
