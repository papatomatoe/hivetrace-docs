# Архитектура

На высоком уровне HiveTrace состоит из:

- **SDK/интеграции** в вашем приложении (отправка `input`/`output`)
- **Сервиса обработки** (валидация, применение политик, обогащение)
- **Хранилища и аналитики** (поиск/фильтры/агрегации)
- **Панели управления** (приложения, политики, алерты, пользователи)

## Поток данных (упрощенно)

1. Ваше приложение отправляет в HiveTrace событие `input` (сообщение пользователя, метаданные, файлы при необходимости).
2. HiveTrace применяет guardrail/кастомные политики и вычисляет результаты мониторинга.
3. Ваше приложение отправляет `output` (ответ модели), чтобы связать цепочку и получить полную наблюдаемость.

:::tip
Для практического старта см. **Quickstart** — там есть минимальный рабочий пример.
:::
