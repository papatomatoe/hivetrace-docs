## Системные пользователи

Страница **«Системные пользователи»** предназначена для управления учетными записями пользователей, имеющих доступ к административной панели HiveTrace. К таким пользователям относятся сотрудники, работающие с мониторингом, настройками и интеграцией платформы.

В этом разделе вы можете:

* просматривать список системных пользователей;
* создавать новые учетные записи;
* удалять пользователей;
* деактивировать учетные записи без полного удаления доступа.

![Страница "Системные пользователи"](/img/system_users_page.png)

### Таблица системных пользователей

На странице отображается таблица со следующими столбцами:

| Поле             | Описание                                |
| ---------------- | --------------------------------------- |
| Email            | Адрес электронной почты пользователя    |
| Имя пользователя | Отображаемое имя пользователя           |
| Роль             | Роль пользователя в системе             |
| Активен          | Статус учетной записи                   |
| Последний вход   | Дата и время последнего входа в систему |

### Добавление системного пользователя

Чтобы создать нового системного пользователя, нажмите кнопку **«Добавить нового пользователя»**.
В открывшемся модальном окне необходимо заполнить следующие поля:

* **Имя пользователя**
* **Email**
* **Роль**
* **Пароль**

После сохранения пользователь получит доступ к системе в соответствии с назначенной ролью.

### Роли пользователей

В HiveTrace предусмотрены две роли системных пользователей:

* **Администратор** — обладает полным доступом ко всем возможностям системы. Администратор может управлять приложениями, политиками, оповещениями, просматривать пользовательские сессии и аналитику, а также создавать, удалять и администрировать системных пользователей.

* **Разработчик** — роль, предназначенная для интеграции приложений с HiveTrace. Пользователь с этой ролью может:

  * просматривать оповещения;
  * создавать приложения;
  * выпускать API-токены для интеграции.

  При этом разработчику **недоступен просмотр пользовательских сессий** (он не видит сообщения пользователей и ответы моделей) и **управление системными пользователями**. Роль предоставляет только минимально необходимые права для подключения и эксплуатации интеграции.
