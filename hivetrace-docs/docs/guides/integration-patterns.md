# Интеграционные паттерны

## Паттерн 1: Middleware вокруг вызова модели

Отправляйте `input` перед вызовом модели и `output` сразу после получения ответа.

## Паттерн 2: Асинхронная отправка (когда важна латентность)

Если основной сценарий чувствителен к латентности, отправляйте события асинхронно, но сохраняйте корреляцию через `session_id`/`user_id`.

:::warning
Асинхронность усложняет гарантии доставки. Для критичных сценариев продумайте retry/backoff и мониторинг ошибок SDK.
:::
